put-up

  • Author: Magnus Kronnäs (krm)
  • Language: Bash
  • Dependencies: cURL
  • Download: download/put-up

Upload files easily from the CLI. Uploads to Hyperboria (h.nup.pw) by default.

Usage: 

      put-up yourFileHere.ext
Example: 
      $ put-up yourFileHere.ext
      [12:47:06] Uploading:  yourFileHere.ext 
      ############################################# 100.0%
      [12:47:07] Done:  put-up -> http://h.nup.pw/hash.ext

nupup

  • Author: William Wennerström (willeponken)
  • License: GPL-3.0
  • Dependencies: cURL
  • Language: Shell
  • Download: download/nupup

Single or multiple file upload, with Internet (default) and Hyperboria support.

Usage: 

      nupup [-a http(s)://address] [-h] file ..

            -a {http(s)://address}  Upload address.
            -h                      Use the Hyperboria address (h.nup.pw).
            file ..                 A list of file names.
Example: 
      $ nupup yourFileHere0.ext yourFileHere1.ext
      ################################### 100.0%
      yourFileHere0.ext: https://nup.pw/hash.ext
      yourFileHere1.ext: https://nup.pw/hash.ext

gonup

  • Author: oniichaNj (oniichan@mrowr.me)
  • License: GPL-3.0
  • Dependencies: None
  • Language: Go
  • Source download: download/gonup.go
  • AMD64 binary download: download/gonup.bin

Golang file uploader for nup.pw, using only standard libraries. Supports multiple files in sequence. No hyperboria support to date.
Works on any system to include a Golang compiler, such as Microsoft® Windows™, Plan 9, OS X and of course GNU/Linux.

Compiling: 

      go build gonup.pw
Usage: 
      gonup file1 file2 ... fileN
Example: 
      $ gonup *
      file.jpg uploading.
      https://nup.pw/hash.jpg
      foo.tex uploading.
      https://nup.pw/hash.tex

nupload

  • Author: pinku (pinku@pinku.pw)
  • License: GPL-3.0
  • Dependencies: cURL, emacs
  • Language: Emacs Lisp
  • Download: download/nupload.el

Emacs Lisp nup.pw client. Can upload both files and buffers over both the Internet and Hyperboria.

Installing:

  • Add nupload.el to somewhere in your load-path
  • (require 'nupload)
Usage: 
      
M-x nupload RET
 or 
M-x nupload-file
